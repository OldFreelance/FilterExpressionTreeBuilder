Описание задачи

Необходимо реализовать статический класс FilterExpressionTreeBuilder с единственным публичным статическим методом Build. Метод Build формирует LINQ выражение. Сигнатура класса описана в проекте ru.ocltd.linq.

Сформированное выражение будет использоваться в качестве делегата (аргумент predicate) в методе Where, реализаций интерфейса System.Linq.IQueryable.

Требования к реализации

Класс FilterExpressionTreeBuilderTests содержит набор тестов, формализующих обобщенные требования к реализации. Класс находится в проекте ru.ocltd.linq.test.

Набор тестов, аналогичный указанному в классе FilterExpressionTreeBuilderTests должен выполняться не только для типа SampleEntity, но и для любого пользовательского типа данных, поля которого реализованы с использованием следующих типов данных:

  • Int
  • String
  • Decimal
  • Boolean
  • DateTime
  • Guid

Основные требования к оформлению кода

Реализованный код должен минимально соответствовать следующему набору правил:

  • Для удобочитаемости кода имена методов и переменных должны быть содержательными.
  • Имена методов должны описывать выполняемую операцию.
  • Реализация любого метода должна содержать от 3 (трех) до 5 (пяти) строк.
  • Любой метод должен содержать не более 2 (двух) входных аргументов.
  • Любой метод должен выполнять одну операцию.

С полным набором требований к оформлению кода можно ознакомиться в книге Роберта Мартина "Идеальный код", издательство "Питер", 2010.
